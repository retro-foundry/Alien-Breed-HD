# Alien Breed 3D I (PC Port)

A C port of Alien Breed 3D I (Amiga), translating the original 68000 assembly into C and rendering with SDL2 on Windows, Linux, and macOS.

The project keeps Amiga data formats and game behavior where possible, while running natively on modern platforms.

![Alien Breed HD Image](docs/alien.jpg)

---

## Repository layout

- Root: PC port source and build system (`src/`, `tools/`, `CMakeLists.txt`).
- `build/data/`: Build-time staged data generated by CMake.
- `amiga/`: Original Amiga assembly sources and source-asset directories used by build sync steps.

---

## What this project includes

- Original Amiga 68000 source for reference (`amiga/`).
- Native C reimplementation of gameplay/rendering/runtime systems (`src/`).
- SDL2-based platform layer for display, input, and audio output.

No Amiga binary is executed by this port.

---

## Game data and build pipeline

Game data is staged by CMake under `build/data/`, then copied beside the built executable (`$<TARGET_FILE_DIR:ab3d1>/data/...`).

Configure-time staging:

- `amiga/pal` and `amiga/data/pal` -> `build/data/pal` (used to generate `src/sprite_palettes_data.h`)

Post-build staging/copies:

- `amiga/vectorobjects` -> `build/data/vectorobjects` -> output `data/vectorobjects`
- `amiga/data/includes` -> `build/data/includes` -> output `data/includes`
- `amiga/data/levels` -> `build/data/levels` -> output `data/levels`
- `amiga/data/gfx` -> `build/data/gfx` -> output `data/gfx`
- `amiga/pal` + `amiga/data/pal` -> `build/data/pal` -> output `data/pal`
- `amiga/data/helper` -> `build/data/helper` -> output `data/helper`
- `amiga/texturemaps` -> `build/data/texturemaps` -> output `data/texturemaps`

Sound conversion pipeline:

1. `amiga/sounds` is synced to `build/data/sounds`.
2. `tools/raw_to_wav.py --sounds-dir build/data/sounds` generates `.wav` files:
   top-level Amiga SFX are treated as raw PCM, and `sounds/mt/*.mt` tracker
   modules are rendered to plain `*.wav` names (for example `InGame.mt` -> `ingame.wav`) via ffmpeg/libopenmpt.
3. `build/data/sounds` is copied beside the executable.

Notes:

- Runtime loads data from executable-local `data/` and exits if required assets are missing.
- Runtime still supports raw Amiga sound files; `.wav` generation is build convenience.
- If source assets are not present in your checkout, provide your own legally obtained data.

## Build

### Requirements

- CMake 3.16+
- C11 compiler (MSVC, GCC, or Clang)
- Python 3 (used by asset helper scripts)
- Network access during first configure (SDL2 is fetched via CMake FetchContent)

### Commands

From repository root:

```bash
mkdir build
cd build
cmake ..
cmake --build . --config Release
```

Typical outputs:

- Windows: `build/Release/ab3d1.exe`
- Linux/macOS: `build/ab3d1`

---

## Run

Run from the build output location (or repo root if you prefer):

```bash
# Windows
build/Release/ab3d1.exe

# Linux/macOS
./build/ab3d1
```
